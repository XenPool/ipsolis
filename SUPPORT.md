# Getting Support

Pick the right channel so your request reaches the right place quickly.

| I want to… | Go to |
|---|---|
| Ask "how do I…?", get help, share my setup | [Discussions → Q&A](https://github.com/XenPool/ipsolis-community/discussions/categories/q-a) |
| Suggest an early-stage idea or give feedback | [Discussions → Ideas](https://github.com/XenPool/ipsolis-community/discussions/categories/ideas) |
| Report a reproducible bug | [Open a bug report](https://github.com/XenPool/ipsolis-community/issues/new?template=bug_report.yml) |
| Request a concrete feature | [Open a feature request](https://github.com/XenPool/ipsolis-community/issues/new?template=feature_request.yml) |
| Report a security vulnerability | **Privately** — see [SECURITY.md](SECURITY.md). Never a public issue. |
| Read the documentation | [ipsolis.com/docs](https://ipsolis.com/docs) |

## Community vs. commercial support

- **Community Edition (AGPL-3.0)** is supported by the community through GitHub
  Discussions and Issues. There is no guaranteed response time — but we read
  everything, and maintainers participate actively.
- **Pro Edition & commercial support** — SLA-backed support contracts,
  on-prem / air-gapped deployment help, security reviews, and consulting are
  available from XenPool GmbH. Contact
  [sales@xenpool.de](mailto:sales@xenpool.de) or visit the
  [shop](https://ipsolis.com/shop).

## Before you post

- Search existing issues and discussions first.
- Include your version (`VERSION` file), edition, and how you deploy.
- **Never paste secrets, license files, or unredacted audit logs / personal data.**
